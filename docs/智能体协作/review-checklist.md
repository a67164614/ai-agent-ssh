# 审查清单

## 规格审查

- 是否覆盖用户原始目标。
- 是否遵守 PRD 的范围和非目标。
- 是否处理错误场景和边界条件。
- API 字段、部署计划 JSON、命令安全规则是否与文档一致。
- 前端是否提供用户可感知的状态反馈。
- 是否更新相关中文文档。

## 后端审查

- 是否有对应 pytest。
- 是否避免信任 AI 原始输出。
- 危险命令是否默认拦截。
- 涉及凭据时是否避免明文日志。
- 数据模型是否为后续 Agent 模式预留扩展点。

## 前端审查

- 菜单和按钮是否有实际交互。
- 接口调用是否显示 loading、成功和失败状态。
- 文案是否中文且清晰。
- 布局在窄屏下是否可用。
- 是否有交互测试或构建验证。

## 部署审查

- Dockerfile 层缓存是否合理。
- Compose 端口、数据卷、环境变量是否清楚。
- `.env` 和密钥是否未提交。
- 文档是否说明更新和回滚方式。

## 交付审查

- 是否运行后端测试。
- 是否运行前端测试和构建。
- 是否说明未运行 Docker 验证的原因。
- 是否记录遗留风险。
