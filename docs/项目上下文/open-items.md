# 遗留事项

## Multica 绑定

- 缺少 `workspaceId` 和 `projectId`。
- 用户已要求当前不处理 Multica，本仓库使用本地中文文档记录需求拆解、执行和交付。

## 产品增强

- 增加更完整的部署回滚策略，例如保留发布版本、回滚命令模板和失败自动停止策略。
- 增加生产级后台任务队列，让长时间部署任务和日志流与 HTTP 请求生命周期解耦。
- 增加多用户、角色权限、审计筛选和 token 过期策略。
- 将前端 `App.tsx` 拆分为 API client、类型定义和独立页面组件，降低维护成本。
- 增加更细的终端权限控制，例如按服务器、命令白名单和操作人维度限制。

## 部署体验

- 当前 Ubuntu 更新流程可以使用 Git 拉取，也可以本地打包上传后执行 `docker compose up -d --build`。
- 可增加 GitHub Actions 构建 GHCR 镜像，减少服务器本地构建耗时。
- 可增加数据库正式迁移脚本，替代 SQLite 轻量升级逻辑。
