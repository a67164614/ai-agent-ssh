# Milestone 1：基础面板 PRD

## 1. 背景

当前项目已经具备 FastAPI 后端骨架、React 运维面板骨架、命令安全检查和部署计划校验能力，但还不能真实管理 AI 中转站和服务器。PRD 的后续 Web 终端、AI 查询、服务分析和部署执行都依赖这些基础能力。

本阶段聚焦“面板可以真实保存配置、管理服务器、测试连接、展示服务器基础信息”。

## 2. 目标

- 支持单管理员初始化和登录。
- 支持页面管理 AI 中转站，API Key 加密保存，只展示掩码。
- 支持测试 AI 中转站连接，拉取模型或手动添加模型。
- 支持服务器新增、编辑、删除、列表和详情。
- 支持 SSH 凭据加密保存，响应中不泄露明文或密文。
- 支持 SSH 连接测试和服务器基础快照。
- 支持基础审计日志，记录关键配置和服务器操作。

## 3. 用户角色

- 管理员：系统第一版唯一用户，可管理 AI 中转站、服务器和基础配置。

第一阶段不实现运维用户、多用户权限和 RBAC。

## 4. 范围

本阶段包含：

- 首次管理员初始化。
- 登录、登出、当前用户查询。
- AI Provider CRUD。
- AI Model 拉取、手动新增、启用/禁用、删除。
- 设置默认 AI Provider。
- AI 连接测试。
- Server CRUD。
- SSH 连接测试。
- 服务器基础快照采集。
- 首页和设置页接真实 API。

本阶段不包含：

- Web SSH 终端。
- AI 生成命令和聊天。
- 服务包上传。
- 项目分析。
- 自动部署执行。
- 回滚。
- GitHub Actions / GHCR 发布。
- Multica 绑定或 issue 同步。

## 5. 业务规则

- 首次启动无用户时允许初始化管理员；已有用户后禁止再次初始化。
- 登录成功后前端保存访问令牌；后端受保护接口需要认证。
- `CREDENTIAL_SECRET` 用于加密 AI API Key、SSH 密码和 SSH 私钥。
- API Key 保存后再次读取只返回掩码，不返回明文或密文。
- SSH 凭据保存后列表和详情均不返回明文或密文。
- 同一时间只有一个默认启用 AI Provider。
- AI Provider 的 Base URL 必须填写到 `/v1`，不能包含 `/chat/completions`。
- `/models` 拉取失败时允许手动添加模型 ID。
- SSH 测试成功后服务器状态更新为 `online`，失败更新为 `offline`，并记录失败原因。
- 服务器快照记录操作系统、内核、CPU、内存、磁盘和 IP 信息。
- 关键操作写入审计日志。

## 6. 接口和字段影响

| 模块 | 接口/字段 | 变化 | 说明 |
| --- | --- | --- | --- |
| Auth | `POST /api/auth/init` | 新增 | 首次管理员初始化 |
| Auth | `POST /api/auth/login` | 新增 | 登录并返回 token |
| Auth | `POST /api/auth/logout` | 新增 | 前端清理 token，后端可先返回成功 |
| Auth | `GET /api/auth/me` | 新增 | 查询当前用户 |
| AI Provider | `/api/ai-providers*` | 新增 | Provider CRUD、测试、模型管理 |
| Server | `/api/servers*` | 新增 | 服务器 CRUD、SSH 测试、快照 |
| 数据模型 | `AiProvider` | 补字段 | `last_test_status`、`last_test_message`、`last_test_at` |
| 数据模型 | `ServerSnapshot` | 新增 | 保存服务器资源快照 |
| 数据模型 | `AuditLog` | 使用 | 记录关键操作 |

## 7. 错误和边界场景

- 重复初始化管理员。
- 登录用户名或密码错误。
- 未登录访问受保护接口。
- Base URL 格式错误。
- AI API Key 无效。
- `/models` 接口不可用。
- 服务器 SSH 连接失败。
- SSH 认证失败。
- 服务器快照命令输出为空或格式不兼容。
- `CREDENTIAL_SECRET` 未在生产环境修改。

## 8. 验收标准

- [ ] Docker 启动后，无用户时可以初始化管理员。
- [ ] 管理员可登录、登出、刷新后保持基本会话状态。
- [ ] 可以新增 AI 中转站，API Key 加密保存，详情只展示掩码。
- [ ] 可以测试 AI 连接并展示成功或失败原因。
- [ ] 可以拉取模型列表；失败时可以手动新增模型。
- [ ] 可以设置默认 Provider 和默认模型。
- [ ] 可以新增服务器，SSH 凭据加密保存。
- [ ] 服务器列表不返回密码、私钥或密文字段。
- [ ] 可以测试 SSH 连接并更新在线状态。
- [ ] 可以采集服务器基础快照并在列表/详情展示。
- [ ] 后端 pytest 通过。
- [ ] 前端 Vitest 和 build 通过。
- [ ] 文档和项目上下文已更新。
